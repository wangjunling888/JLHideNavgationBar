# JLHideNavigationBar

#### 如何用?

1. 通过cocopods使用

   podfile文件中加入如下代码

   ```pod 'JLHideNavigationBar'```

2. 可以直接下载代码仓库到本地, 添加到项目中

   

#### 有什么用?

项目中经常遇到需要隐藏导航的页面,  之前直接使用 **setNavigationBarHidden:animated:** 方法去隐藏, 这样做有一个问题, 就是再离开页面的时候把导航栏添加上(viewWillDisappear方法中设置),  正常情况下这样做是没问题的, 但是遇到如下情况可能就会有问题

A(无导航页面) -> B(无导航) -> C(有导航)

在页面A的viewWillDisappear中将导航显示, 在B的viewWillAppear中将导航隐藏, 在从B返回到A的过程中也是先显示后隐藏, 这个时候就会出现问题,  在从B返回A的时候会出现导航栏闪一下的问题, 因为从B到A实际上导航栏是不需要显示的,  那么如何解决问题呢, 首先想到的肯定是在viewVillDisappear中如果能判断上个页面或者下一个页面是否需要导航栏这样不就可以解决了吗?  然后顺着思路就开始将A页面是否需要显示带给B, C的状态也应该给B,  这样导致另一个问题出现, 就是B需要知道所有和B有关联页面的导航状态, 这么写代码肯定是反人类的, 因为ABC可能是三个不同的人写的, 在实际项目中协作这个事情将会变得困难. 

通过以上分析如果我ABC每个控制器都有个属性来控制自己的导航显示与否, 而且需要在适合的地方自动调用一次, 不就完美解决了吗?  

所以自然而然的就出现了此库,  给每个控制器添加一个属性, 并且在适合的时候调用, 实现效果就是, 因为默认导航是不隐藏导航栏的, 所以只需要在需要隐藏导航栏的控制器中设置隐藏与否即可完美解决!
